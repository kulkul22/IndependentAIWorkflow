# Design specification checklist

- Product user and top journeys are explicit.
- Page hierarchy and navigation are defined.
- At least two visual directions were considered.
- Chosen direction includes rationale and reference evidence.
- Color, typography, spacing, radius, border, shadow, and motion tokens are concrete.
- Desktop and mobile layouts exist for every primary page.
- Loading, empty, error, success, disabled, destructive, focus, hover, and overflow states are defined.
- Tables and charts include dense-data and empty-data behavior.
- Contrast target is WCAG AA or stronger.
- Each visible promise has a testable `UI-ACC-*` criterion.
- No frontend implementation ticket can be marked done without its acceptance IDs and visual evidence.
