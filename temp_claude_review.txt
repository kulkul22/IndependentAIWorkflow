## Edge Cases & Improvements

1. **ChromaDB corruption on crash**: If the daemon dies mid-upsert, the SQLite DB may corrupt. Wrap sync in try/except and add a startup integrity check.

2. **Batch updates**: Instead of upserting all vault files on every trigger, track which specific files changed and upsert only those. The current `glob` on line 33 re-processes the entire vault.

3. **Graceful shutdown**: On `KeyboardInterrupt` (line 60), flush any pending ChromaDB writes before `observer.stop()`.

4. **Remove `FileLock`**: Once you move to a single long-running process, the file lock becomes unnecessary—remove it to eliminate a deadlock vector.

5. **Windows-specific**: ONNX runtime can be slower on Windows. Consider using `chromadb`'s built-in `ONNXMini_LM_L6_V2` with explicit `device="cpu"` to avoid GPU driver issues.

## Better Alternative?
For a production RAG pipeline on Windows, consider:
- **Event-driven queue** (e.g., `queue.Queue`) instead of watchdog debounce.
- **Incremental sync**: Hash file contents; skip upsert if unchanged.
