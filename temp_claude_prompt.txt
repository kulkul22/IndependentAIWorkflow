### Current Implementation

**javis_daemon.py**
```python
import os
import sys
import time
import subprocess
from watchdog.observers import Observer
from watchdog.events import FileSystemEventHandler

PROJECT_ROOT = os.path.abspath(os.path.join(os.path.dirname(__file__), '..'))
VAULT_DIR = os.path.join(PROJECT_ROOT, 'brain', 'vault')
SYNC_SCRIPT = os.path.join(PROJECT_ROOT, 'scripts', 'brain_sync.py')

class VaultChangeHandler(FileSystemEventHandler):
    def __init__(self):
        super().__init__()
        self.last_sync = 0
        self.debounce_seconds = 5 # Prevent multiple syncs in a short time

    def on_modified(self, event):
        if not event.is_directory and event.src_path.endswith('.md'):
            self.trigger_sync()

    def on_created(self, event):
        if not event.is_directory and event.src_path.endswith('.md'):
            self.trigger_sync()

    def trigger_sync(self):
        current_time = time.time()
        if current_time - self.last_sync > self.debounce_seconds:
            self.last_sync = current_time
            subprocess.run([sys.executable, SYNC_SCRIPT], check=True)

# daemon loop...
```

### Diagnosis
The system crashes because every time a `.md` file is saved, `javis_daemon.py` calls `brain_sync.py` as a new subprocess. `brain_sync.py` initializes ChromaDB and `DefaultEmbeddingFunction`, which loads the `all-MiniLM-L6-v2` ONNX model into RAM. Doing this every 5 seconds under heavy editing consumes immense CPU and RAM, causing the OS to thrash and the IDE to crash. It also locks files and interacts poorly with IDE file watchers.

### Proposed Solution
1. **Long-running Process**: Import `sync_vault` logic directly into `javis_daemon.py`. Initialize `chromadb` and `DefaultEmbeddingFunction` ONLY ONCE globally at startup, and pass the `collection` instance to the sync function. This keeps the model in RAM rather than reloading it on every trigger.
2. **Increase Debounce**: Increase the debounce time to 30-60 seconds, or implement a batching queue, to avoid aggressively syncing during active writing.
3. **Restrict Watchdog Scope**: Ensure the watchdog strictly listens only to `VAULT_DIR` and ignores other `.md` files auto-generated by AI agents elsewhere in the project to prevent infinite sync loops.
